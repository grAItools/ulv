## Install `uv`

[`uv`](https://docs.astral.sh/uv/) — fast Python package & project manager.

The canonical, automated bootstrap is
[`.agents/hooks/ensure-toolchain.sh`](../.agents/hooks/ensure-toolchain.sh): it
installs `uv` if it is missing, and is exactly what the Claude Code
`SessionStart` hook (and a Claude Code on the web setup script) run. From the
repo root:

```sh
sh .agents/hooks/ensure-toolchain.sh
```

What that does under the hood (and how to install by hand):

```sh
# Standalone installer (macOS / Linux):
curl -LsSf https://astral.sh/uv/install.sh | sh

# Or via pipx / brew / your OS package manager — see
# https://docs.astral.sh/uv/getting-started/installation/

uv --version   # verify
```

After install: `uv sync` materialises the project environment from
`pyproject.toml` and creates/updates `uv.lock` (or uses it when present).

## Activate the language toolchain

If this repo pins language versions in a pin file (neither file is
generated by the template, so add one as part of project setup), use
a runtime version manager to install the pinned versions. Pick **one**
pin file:

- **`.tool-versions`** — read by both `asdf` and `mise`. Choose this
  if you want either tool to work out of the box.
- **`mise.toml`** — read only by `mise`, but supports richer config
  (env vars, tasks, etc.). Choose this if you've committed to `mise`.

Then pick a manager:

### Option A — [`mise`](https://mise.jdx.dev/) (recommended)

```sh
# Install mise (one-time, machine-level):
curl -fsSL https://mise.run | sh
# Or via your OS package manager — e.g. `brew install mise` —
# see https://mise.jdx.dev/getting-started.html for all options.

# Activate in your shell (add to ~/.bashrc / ~/.zshrc / config.fish):
eval "$(mise activate bash)"   # or: zsh / fish

# Inside the repo:
mise install                   # installs every pinned tool
mise current                   # show resolved versions
```

### Option B — [`asdf`](https://asdf-vm.com/)

```sh
# Install asdf (macOS):  brew install asdf
# Or follow https://asdf-vm.com/guide/getting-started.html

# Add language plugins (one-time, per language):
asdf plugin add python         # then: asdf plugin add nodejs, etc.

# Inside the repo (reads .tool-versions):
asdf install                   # installs every pinned version
asdf current                   # show resolved versions
```

Per-language version managers (`pyenv`, `nvm`, `rustup`, `rbenv`, …)
also work — use the same versions you've recorded in your chosen pin
file (`.tool-versions` or `mise.toml`); they typically read their own
per-language pin files (`.python-version`, `.nvmrc`, …) so keep those
in sync with the main pin file if you go this route.
